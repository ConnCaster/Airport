# Airport

### 1. Ground Control (порт 8081)

```
GET /v1/land_permission?flightId=...
    Запрос разрешения на посадку (Board -> GroundControl)
    
GET /v1/takeoff_permission?flightId=...    
   Запрос разрешения на взлет (Board -> GroundControl)
    
POST /v1/flights/{flightId}/landed
    Информирование о посадке в RE-1 ноду карты аэропорта (Board -> GroundControl)
      
POST /v1/flights/{flightId}/tookoff
    Информирование о фактическом взлете (Board -> GroundControl) 
      
POST /v1/vehicles/init
    Информирование о всех созданных машинках FollowMe, чтобы отрисовать их на карте (FollowMe -> GroundControl)
     
GET /v1/map/followme/path?flightId=...
    Запрос маршрута для сопровождения севшего рейса до парковки (FollowMe -> GroundControl)
    
GET /v1/map/followme/permission?flightId=...
    Запрос разрешение на начало сопровождения севшего рейса до парковки (FollowMe -> GroundControl)

POST /v1/map/traffic/enter-edge
    Информирование о входе в ребро по маршруту от посадки до парковки самолета машинкой FollowMe (FollowMe -> GroundControl)
    
POST /v1/map/traffic/leave-edge
    Информирование о выезде из ребра по маршруту от посадки до парковки самолета машинкой FollowMe (FollowMe -> GroundControl)

POST /v1/vehicles/followme (status=arrivedParking)
    Информирование о статусе выполнения доставки самолета до парковки машинкой FollowMe в процессе выполнения (FollowMe -> GroundControl)

POST /v1/followme/mission/completed
    Информирование о выполнении доставки самолета до парковки машинкой FollowMe в итоге (FollowMe -> GroundControl)

GET /v1/visualizer/snapshot
    Получение актуального состояния системы аэропорта: карта с занятыми кем-то ребрами или узлами (Visualizer -> GroundControl)

GET /v1/visualizer/events?since=...
    Получение полного лога событий перемещения машинки FollowMe (в перспективе и всех других транспортов по аэропорту) с определенного момента времени (Visualizer -> GroundControl) 

GET /health
    Проверка, что сервис запустился (* -> GroundControl)
```

### 2. Information Panel (порт 8082)

```
POST /v1/flights/init
    Инициализация всех рейсов, которые запланированы к посадке с указанием времени ожидания
    P.S.: лучше заменить на чтение из БД

GET /v1/flights/{flightId}?ts=...
    Запрос, ожидаем ли мы посадку рейса в определенное время (GroundControl -> Information Table)

POST /v1/flights/status
    Изменение статуса рейса (ожидаем, сел, взлетел и так далее) (GroundControl -> Information Table)

GET /v1/flights
    Запрос всех запланированных рейсов (GroundControl -> Information Table)
```

### 3. FollowMe (порт 8083)

```
POST /v1/vehicles/init
    Запрос на создание машинок FollowMe на спецпарковке FS-1 (GroundControl -> FollowMe)

GET /v1/vehicles/hasEmpty
    Запрос, есть ли свободные машинки FollowMe: будет ли кому доставить самолет до парковки (GroundControl -> FollowMe)

POST /v1/vehicles/reserve
    Резервирование машинки FollowMe для обслуживания самолета (GroundControl -> FollowMe)

POST /v1/vehicles/release
    Отмена резервирование машинки FollowMe (GroundControl -> FollowMe)

* GET /v1/vehicles (не используется)
    Получение информации о всех машинках FollowMe (GroundControl -> FollowMe)

GET /health
    Проверка, что сервис запустился (* -> FollowMe)
```

Списки атрибутов для POST-запросов искать в коде.
В директории scripts можно вызывать скрипы для отдельных API-ручек.

Цель сборки *Board* компилит бинарь, который можно использовать для тестирования посадки самолета.

### Порядок запуска сервисов
1. Information Panel
2. Ground Control
3. FollowMe
4. Board

## Порядок действий 
### I. Инициализация системы
1. [x] При запуске InformationPanel происходит чтение информации о самолетах из БД. Можно не читать, тогда по необходимости отправлять запросы в БД.
2. [x] После запуска сервиса GroundControl он запрашивает у InformationPanel название рейса, парковочное место
   и может что-то еще для все стоящих в аэропорту самолетов.
   При этом он помещает полученные самолеты в узлы карты - парковочные места (добавляет в snapshot для Visualizer)
3. *GroundControl, получив информацию о самолетах на стоянках, отправляет запрос в Board для создания отдельных сущностей самолетов (вероятно, потоков), 
   которые будут заниматься запросами на свое обслуживание через HandlingSuperviser(заправка, пассажиры, питание).
4. *GroundControl, получив информацию о самолетах в воздухе (они же когда то запланированные к посадке), отправляет запрос в Board для создания отдельных сущностей самолетов (вероятно, потоков), 
   которые будут запрашивать у GroundControl разрешение на посадку, которое GroundControl будет получать у InformationPanel
5. [x] GroundControl, получив и обработав информацию о самолетах, отправляет на сервис FollowMe запрос на инициализацию машинок.
   Эти машинки читаются из БД и информация о них отправляется в GroundControl, который наносит их на карту и добавляет в snapshot для Visualizer.

### II. Посадка
1. Самолет в воздухе непрерывно спрашивает GroundControl о возможности посадки.
2. GroundControl:
     - спрашивает InformationTable об ожидании рейса, 
     - проверяет, что посадочная полоса свободна, 
     - спрашивает FollowMe о наличии свободных машиной для буксировки после посадки
     - ищет свободную парковочную площадку
     - бронирует машинку
     - РАЗРЕШАЕТ или ОТКАЗЫВАЕТ в посадке
3. Забронированная FollowMe машинка опрашивает GroundControl, можно ли начать движение.
4. Если разрешает - Board оказывается в точке посадки RE-1, и информирует GroundControl о посадке. 
5. *Получив информирование от Board о посадке GroundControl отправляет информирование на InformationTable.
6. Получив информирование от Board о посадке GroundControl разрешает отправку машинки FollowMe за рейсом.
7. Машинка уезжает.
8. О своем перемещении по карте машинка FollowMe сообщает постоянно GroundControl
9. О доставке к парковочному месту самолета машинка FollowMe сообщает статус GroundControl
10. О возвращении на место стоянки машинок FollowMe они также сообщают статус GroundControl

### III. Наземное обслуживание (Поверхностно, но не очень)
1. *GroundControl, получив информирование о доставке Board до парковочного места от FollowMe, отправляет соответствующий статус на InformationTable
2. *Board проверяет уровень топлива и если оно ниже фиксированного порога - отправляет в HandlingSuperviser запрос на дозаправку с объемом необходимого топлива 
3. *HandlingSuperviser, получив запрос на дозаправку от Board, инициирует отправку машинки Refueler (там что-то происходит дальше с топливом).
3. *?Получив уведомление о посадке конкретного рейса, InformationTable отправляет запрос на HandlingSuperviser о необходимости выполнения цепочки задач:
    - отправить автобус за пассажирами от самолета до гейта
    - отправить забрать с самолета багаж
    - отправить кейтеринг
    - ...
    - отправить автобус с пассажирами от гейта до самолета
4. *? Каждая машинка из п.12 должна отправлять на конкретный рейс информацию о том, кого и сколько она привезла и увезла, чтобы самолет владел информацией о себе
5. *Каждая машинка из п.12 должна отправлять свои статусы HandlingSuperviser.
6. *Когда цикл обработки одного рейса завершен - HandlingSuperviser отправляет на самолет информирование о завершении наземного обслуживания.
7. *Board готов взлетать.

### IV. Взлёт
1. *Board, получив информацию о завершении наземного обслуживания от HandlingSuperviser начинаем через таймауты запрашивать разрешение на взлет у GroundControl
2. *GroundControl запрашивает у InformationTable разрешение на взлет
3. *Получив отказ, отказывает Board
4. *Получив разрешение, разрешает Board взлет
